# Remote access — dev PC → phones (Termux SSH over Tailscale)

Run commands and upload files from the PC without pasting into Termux.

## Prerequisites

1. Tailscale connected on PC and both phones.
2. On each phone in Termux (once):

```bash
pkg install openssh
passwd                    # you chose 123456
sshd                      # port 8022
whoami                    # phone-a: u0_a360, phone-b: u0_a138
```

3. On PC — copy secrets and run setup:

```powershell
cd C:\workspace\Ezrababait-2023\phone-lab
copy mesh.secrets.env.example mesh.secrets.env
# edit mesh.secrets.env → PHONE_SSH_PASSWORD=123456
.\scripts\setup-phone-ssh.ps1
```

This generates `~/.ssh/phone-lab`, installs the public key on both phones, and adds `~/.termux/boot/start-sshd.sh`.

**After setup, password is not needed** — the agent uses the SSH key.

## Daily usage

```powershell
# run one command
.\scripts\remote-exec.ps1 phone-b "curl -s http://127.0.0.1:4010/public/api/agents/health/ready"

# tail log
.\scripts\remote-exec.ps1 phone-b "tail -30 ~/phone-lab/logs/agents-prod.log"

# native ssh (after adding scripts/remote/ssh-config.snippet to ~/.ssh/config)
ssh phone-b whoami
```

## pgAdmin — access phone-b PostgreSQL via SSH tunnel

Recommended way to inspect the `api-agents` database on **phone-b** from the dev PC without exposing port 5432.

### Register -> Server...

**General**

- Name: `phone-b api-agents (SSH)`

**Connection**

- Host name/address: `127.0.0.1`
- Port: `5432`
- Maintenance database: `agents`
- Username: `admin`
- Password: `password123` (or check `~/phone-lab/packages/api-agents-prod/.env` on phone-b)

**SSH Tunnel**

- Use SSH tunneling: ON
- Tunnel host: `PHONE_B_IP` from `mesh.env` (e.g. `100.103.183.36`)
- Tunnel port: `PHONE_SSH_PORT` from `mesh.env` (default `8022`)
- Username: `PHONE_B_SSH_USER` from `mesh.env` (e.g. `u0_a138`)
- Authentication: Identity file
- Identity file: `C:\Users\<YOU>\.ssh\phone-lab` (generated by `.\scripts\setup-phone-ssh.ps1`)

## Config files

| File | Committed | Purpose |
|------|-----------|---------|
| `mesh.env` | no | Tailscale IPs + SSH users |
| `mesh.secrets.env` | no | `PHONE_SSH_PASSWORD` |

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Connection refused` :8022 | On phone: `sshd` or open Termux |
| `Permission denied` | Re-run `.\scripts\setup-phone-ssh.ps1` |
| `paramiko not found` | `python -m pip install --user paramiko` |
| SSH works, HTTP smoke fails | Tailscale VPN off on phone |

## Security

- SSH is only reachable via Tailscale (100.x.x.x).
- Do not expose port 8022 to the public internet.
- Change the Termux password from `123456` in production-like setups.
